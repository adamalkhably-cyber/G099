Security Notice

This dashboard currently checks the admin username and password directly in the page's JavaScript, which means anyone can view the source and read the credentials. Before this goes live, move login checks to a server endpoint that verifies against the admin_users table's hashed password, and issue a session token instead of trusting the browser.

The "live" login feed on this page is simulated in the browser for demo purposes. To make it real, have your backend write a row to login_log on every user login, and either poll that table every few seconds or push new rows to the dashboard over a WebSocket.
